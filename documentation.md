# Git Task Documentation

## Objective
Manage a DevOps project using Git best practices.

## Steps Performed
1. Initialized a Git repository.
2. Created `main`, `dev`, and `feature/<name>` branches.
3. Used pull requests for merging branches.
4. Added a `.gitignore` file.
5. Created annotated tags.
6. Wrote this documentation in markdown.

## Git Commands Used
```bash
git init
git checkout -b dev
git checkout -b feature/new-feature
git add .
git commit -m "Initial commit"
git push origin feature/new-feature
git tag -a v1.0 -m "First stable release"
```

## Conclusion
This task helped understand Git workflows, merge conflicts, and version control standards.
